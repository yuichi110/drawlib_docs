Coding Best Practices

If your illustration is comple, drawlib’s code will be also complex. On that situation, coding practice is important.

First of all, you need to notice that arranging coordinate and size will happen many times. Actions for easing changing them are drawlib’s coding best practices!!

Consider Which Alignment is the best

Many drawing items are aligned vertically and horizontally. Using horizontal vertical center is best for aligning different type and size objects.

Let’s compare codes. They are almost same but difference in alignment.

image1.py
 1from drawlib.apis import *
 2
 3config(width=100, height=100, grid_only=True)
 4
 5# left bottom align
 6text(
 7    (19, 73),
 8    "Drawlib",
 9    style=TextStyle(size=24, halign="left", valign="bottom"),
10)
11icon_phosphor.heart(
12    (45, 70),
13    width=10,
14    style=IconStyle(halign="left", valign="bottom"),
15)
16circle(
17    (70, 70),
18    radius=5,
19    style=ShapeStyle(halign="left", valign="bottom"),
20)
21
22# center center align
23text(
24    (25, 25),
25    "Drawlib",
26    style=TextStyle(size=24, halign="center", valign="center"),
27)
28icon_phosphor.heart(
29    (50, 25),
30    width=10,
31    style=IconStyle(halign="center", valign="center"),
32)
33circle(
34    (75, 25),
35    radius=5,
36    style=ShapeStyle(halign="center", valign="center"),
37)
38
39save()

Upperside drawing uses Style(halign="left", valign="bottom"). The (x,y) value is not a simple values since I calculate/test positions.

However, bottomside (x,y) values are very easy to understand. They are just pointing center of each items. No calculation and test are required.

Align “center, center” is nice work when you change item size. You don’t need to change coordinate if center of the new size item doesn’t change. You need to change (x,y) position if you use left, bottom alignment.

Use Variable When Having Same Values

At last example, we need to change all y values when we want to change y position of all items. With align center, typically having same y when aligning items horizontally. And also, same x when aligning items vertically. Let’s use variable for coordinations.

In this example, we use coordination variable for holrizontally aligning items.

image2_1.py
 1from drawlib.apis import *
 2
 3width = 100
 4height = 100
 5config(width=width, height=height, grid_only=True)
 6
 7num_items = 3
 8margin_x = width / (num_items + 1)
 9y = height / 2
10
11# center center align
12text((margin_x, y), "Drawlib", style=TextStyle(size=24))
13icon_phosphor.heart((margin_x * 2, y), width=10)
14circle((margin_x * 3, y), radius=5)
15
16save()

Var y is simple. It is a half of height. Var x is little bit complex, but it provides same x margin for items.

You can easily change canvas height and add new items in this code. We will change height 50 and add rectangle.

image2_2.py
 1from drawlib.apis import *
 2
 3width = 100
 4height = 50  # <= CHANGE!!
 5config(width=width, height=height, grid_only=True)
 6
 7num_items = 4  # <= CHANGE!!
 8margin_x = width / (num_items + 1)
 9y = height / 2
10
11# center center align
12text((margin_x, y), "Drawlib", style=TextStyle(size=24))
13icon_phosphor.heart((margin_x * 2, y), width=10)
14circle((margin_x * 3, y), radius=5)
15rectangle((margin_x * 4, y), width=10, height=10)
16
17save()

As you can see, there are only few point of change. In my opinion, you should follow example of y. But exmaple of x makes code complex. If you are not planning to add items, Not using variable and calculation is one choice. We will hard code 25, 50, 75 to x coodinate at first example normally.

New image align the items correctly.

Grouping by Function

Function is useful for calling same operation repeatedly. You can get same benefit at drawlib of course. However, using function for just grouping items is also recommended coding style.

When you group items via function, you will get these benefits.

  • Able to understand abstract image design

  • Able to localize variable such as x,y which doesn’t affects outside of function.

  • Draw same group of items at many places

Normally, we tend to create these functions

  • left(), center(), right()

  • bottom(), center(), top()

At each functions define x and y at first. All items are relative position from the base x, y position. When you want to move group, changing the x, y vars moves all items position.

Create Your Package

When you need to create many images, consistency is important. You can create independent image codes. But we recommend creating package with these reasons.

  • Same style for all images. Changing style code effects all images quickly.

  • Create utility functions on utility code file. It makes your image code simple.

  • Grouping images by chapters and sections
